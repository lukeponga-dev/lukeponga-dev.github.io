## üêõ Submitting a Bug Report

If you find a bug, please follow these steps:

1.  **Check Existing Issues:** Before opening a new issue, please check if a similar bug has already been reported.
2.  **Open a New Issue:** Use the **Bug Report** template if one is provided.
3.  **Provide Detail:** Include as many details as possible:
    * A clear and descriptive title.
    * **Steps to reproduce** the bug.
    * What you **expected** to happen.
    * What **actually** happened.
    * Your operating system and browser version.

## ‚ú® Suggesting an Enhancement

If you have an idea for a new feature or major improvement:

1.  **Open an Issue:** Use the **Feature Request** template if one is provided.
2.  **Describe the Idea:** Clearly explain the problem your enhancement solves and how you propose to solve it.
3.  **Discuss:** Wait for project maintainers to discuss the proposal before beginning development.

## üöÄ Submitting a Pull Request (PR)

Follow these steps for all code and content contributions:

1.  **Fork the Repository** and clone it locally.
2.  **Create a New Branch:** Base your work on the `main` branch and use a descriptive name (e.g., `bug/fix-header-link` or `feature/add-contact-form`).
3.  **Make Changes:** Implement your fix or feature.
4.  **Test Locally:** Ensure the site builds and runs correctly on your machine.
5.  **Commit Your Changes:** Write clear, concise commit messages. A good standard is: `<type>: <subject>` (e.g., `fix: Corrected broken link in footer`).
6.  **Push to your Fork:** Push the changes to your branch on your forked repository.
7.  **Open a Pull Request (PR):**
    * Target the `main` branch of `lukeponga-dev/lukeponga-dev.github.io`.
    * Provide a clear title and description of the changes.
